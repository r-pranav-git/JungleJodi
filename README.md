# 🌲 JungleJodi

JungleJodi is a creative full-stack web platform that reimagines modern dating applications for animals living in a forest ecosystem.  
Instead of human-centric preferences and text-based chats, JungleJodi focuses on ecological compatibility, immersive forest interactions, and playful storytelling.

## 🧠 Application Overview

JungleJodi places animals inside a living forest ecosystem where they can discover compatible companions based on ecological and behavioral traits such as habitat, food habits, and activity patterns.

The platform replaces traditional chat-based communication with imaginative, forest-themed interactions and applies ecosystem-aware matchmaking logic, including predator–prey restrictions. The aim is to deliver a playful, immersive, and story-driven matchmaking experience while maintaining logical consistency within a forest environment.

---

## ✨ Implemented Features

### 🐾 Animal Profile Creation
- Profiles created using species, habitat, food habits, activity time, and personality traits.

### ⚖️ Ecosystem-Aware Matchmaking
- Rule-based compatibility logic.
- Predator–prey restriction to avoid unsafe matches.

### 🌳 Forest-Themed Interaction System
- Interactions such as Tree-top Swing, Riverbank Walk, and Moonlight Hunt.
- No traditional text-based chat system.

### 🍂 Seasonal Theme Modes
- Spring, Monsoon, and Night Forest themes.
- Dynamic UI and mood changes.

### 🎮 Gamified User Experience
- Ambient forest sounds.
- Visual animations and interaction feedback.

### ⚙️ Backend API with MongoDB
- Authentication, animal discovery, and matchmaking APIs.
- Modular and scalable backend architecture.

## ▶️ Setup and Execution Instructions

### Prerequisites
- Node.js (v18 or higher)
- npm

---

### Backend Setup

```bash
cd junglejodi-backend
npm install
node src/index.js
```

Backend runs at:
http://localhost:5000

### Frontend Setup

```bash
cd frontend
npm install
npm run dev
```

Frontend runs at:
http://localhost:5173

## ⚠️ Assumptions, Limitations & Known Issues

The application is designed primarily for demonstration and hackathon purposes.  
Security mechanisms are intentionally lightweight to prioritize rapid development.  
Some interactions are simulated rather than real-time to maintain simplicity and stability.
